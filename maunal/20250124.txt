id 값으로 원하는 위치로 이동하기
  p 태그들 100개 가 있는중, 원래대로 라면 p 들끼리의 구분을 할 수 없으나
  id 라는 속성을 지정하고 a 태그를 사용하여 id 중 하나로, 그리고 #기호를 통해
  해당 id로 이동함

  페이지에 있는 특정한 요소로 이동하고 싶을 때 id 값을 사용한 링크를 넣어주게 되면
  스크롤이 긴 페이지에서도 원하는 정보가 있는 곳으로 빠르게 이동 가능

      form 태그의 속성
        예를 들어 이름과 나이 입력하는 양식 있다면 우리는 키보드로 해당 정보 입력
        이름 폼에는 텍스트를 입력
        나이 폼에는 숫자를 입력 하거나 마우스 휠을 이용한 나이 입력하는

        사용자로 부터 입력 받기위한 양식을 작성하는 태그 전체를 form 이라 함
        그 속성부터 이름 나이 입력요소 제출 초기화 등 정보 제출하기 위한 태그를 포함

      form 내부의 속성
        action : 우리가 이름 / 나이를 입력 했을경우 연결되는 결과 페이지
        method : 가서 보여주도록 method 를 고정 POST/ GET/ PUT / DELETE
          HTTP method 를 의미
        autocomplite : 자동완성속성 default 로 on이기 때문에 꺼 주고 싶으면 off 로 설정

        input 태그 : 종류가 다양해서 n1 에서 작성한 부분만 다룰 예정
          
          1. name : HTML 상으로 특별한 의미는 아니고 서버에서 각 항복을 받을때
          무엇에 대한 값인지 구분 가능하도록 정보를 보내는 용도
          
          2. label : 이상의 input 태그와 유기적으로 연결 되어있음 , input 태그에 부여한 id 속성
          값을 label 태그와 for 속성에서 불러오기 때문
          input-name -> label-for

          3. button : emmet 이용해 목록 확인 했을 경우 제출인지 결과인지 확인 가능

    form 태그 안 요소 그룹화

  기타 <input>
    사용자가 어떤 파일 첨부해서 업로드 하거나 내용 숨기는 것 , 이메일 양식 작성 후 제출하는것 . etc   

    type :  
      file : 파일 첨부시 사용
        accept : 원하는 유형의 파일만 업로드 할 수 있도록 제어.
        multiple : 파일 여러개 한번에 올릴 수 있도록 함
        hidden : 무언가 숨겨져 있는 방식
            사용자로부터 받은 개인정보, 사용자가 알아보면 안되는 등급의 정보 때문
            JavaScript와 합쳐져서 작성하는 편 
        email : email 형식으로 작성하지 않으면 경고창 
          다만 아무 텍스트나 넣고 @만 넣어줘도 제출 되는 등 허술한 부분 있음
          추후 자바스크립트 통해 개선

    input 태그의 공통 속성
      n7commoninputtag.html    

      1) value : placeholder 와 달리 어떤 default 값을 미리 생성하고 싶을때 사용
      2) autofocus : 어떤 페이지가 새로고침 됐을 때 클릭하지 않아도 자동으로 어떤 요소가 선택된 상태 유지하도록 하는 속성
      3) readonly : 읽기만 가능하고 입력값은 결과로 전송됨
      4) disabled : 입력 불가능 및 결과 전송 안됨
      5) required : 반드시 인풋 값을 입력해야하는 속성 // 채워진 속성은 전송됨
